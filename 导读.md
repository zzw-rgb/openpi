# openpi / π0.5 中文导读

> 本仓库英文文档已就地中文化（`README.md`、`docs/`、各 `examples/*/README.md`）；π0.5 核心源码已加中文注释。本文件是唯一总导读，看完即可上手。

---

## 1. 这是什么

openpi 是 Physical Intelligence 的机器人策略框架，核心是 VLA（视觉-语言-动作）模型 **π0 / π0.5**：

- **输入**：多路相机图像 + 机器人状态（关节角、夹爪）+ 一句话指令
- **输出**：未来一段动作序列 `[action_horizon, action_dim]`，直接给机器人执行
- π0 与 π0.5 **共用一套代码**，靠 `pi05` 开关切换。

---

## 2. 两条主线（记住这个就不会迷路）

```
训练：数据集 → data_loader → transforms(归一化+tokenize) → pi0.py::compute_loss(flow matching) → train.py 反传更新
推理：观测   → policy.py::infer(input_transform)         → pi0.py::sample_actions(去噪) → output_transform(反归一化) → 动作
```

---

## 3. 源码按这个顺序读

| 顺序 | 文件 | 看什么 |
|---|---|---|
| 1 | `models/model.py` | 输入输出长啥样：`Observation`、预处理 |
| 2 | `models/pi0_config.py` | π0 / π0.5 的开关 `pi05` |
| 3 | `models/pi0.py` ⭐ | **最核心**：`compute_loss`、`sample_actions`、注意力 mask |
| 4 | `models/tokenizer.py` | π0.5 怎么把 state 塞进语言 token |
| 5 | `transforms.py` | 数据怎么变成模型格式（归一化/tokenize/pad） |
| 6 | `training/config.py` | 一个训练任务怎么定义（找 `pi05_*` 条目） |
| 7 | `policies/policy.py` | 推理主流程；`serving/` 是起服务 |

**读到第 3 步（`pi0.py`）就懂了 80%。**

---

## 4. π0 vs π0.5：三个区别（可拆开用）

| | π0 | π0.5 |
|---|---|---|
| 状态输入 | 连续 state token 进 suffix | 离散成 256-bin 文本 token 走语言通道 |
| 时间注入 | 时间编码与动作拼接过 MLP | **adaRMS** 把时间作条件注入 |
| max_token_len | 48 | 200 |

> 靠 `pi05` 开关。其中"state 是否离散"由独立字段 `discrete_state_input` 控制（默认跟随 `pi05`，可覆盖）。如 `pi05_libero` 设 `pi05=True, discrete_state_input=False`，即用 adaRMS 但两条 state 通路都不走、只靠图像+prompt。

**flow matching 一句话**：训练时在纯噪声和真实动作间插值，让模型学"指向真实动作的速度场"；推理时从纯噪声一步步欧拉积分回真实动作（默认 10 步）。

## 5. 在 Franka 上用（真机，对应 DROID 平台）

> Franka = openpi 里的 **DROID 平台**（Franka Panda + ZED 相机 + VR 遥操）。这是**真机不是仿真**——客户端 `examples/droid/main.py` 用 `droid.robot_env.RobotEnv` 直接驱动真实机械臂。仿真在 `examples/libero`、`examples/aloha_sim`。

### 5.1 部署（双机：GPU 机跑服务 + DROID 笔记本当客户端）

```bash
# ① GPU 机上起策略服务
uv run scripts/serve_policy.py policy:checkpoint \
    --policy.config=pi05_droid --policy.dir=gs://openpi-assets/checkpoints/pi05_droid

# ② DROID 笔记本上装客户端并运行（先改 main.py 里的相机 ID）
cd $OPENPI_ROOT/packages/openpi-client && pip install -e . && pip install tyro
python3 scripts/main.py --remote_host=<server_ip> --remote_port=<port> --external_camera="left"
```
运行后输入一句自由文本指令即可。只用 1 路外部相机 + 1 路腕部相机，延迟约 0.5–1 秒/块属正常。

### 5.2 采集数据（openpi 不含采集器）

用 DROID 平台 VR 遥操采集（录 `.svo`），再转成 openpi 能训的格式：
```bash
# (可选) 下载官方 30 条示范练手
gsutil -m cp -r gs://gresearch/robotics/droid_raw/1.0.1/IRIS/success/2023-12-04 <path>
# 转成 LeRobot 格式
uv run examples/droid/convert_droid_data_to_lerobot.py --data_dir <path>
# 微调 pi05_droid（配置见 config.py 的 pi05_droid_finetune）
uv run scripts/train.py pi05_droid_finetune --exp-name=my_exp --overwrite
```

### 5.3 数据格式（模型的输入输出契约）

**机器人端只需发这 5 样**（`main.py` 就是发这些）：

| 键 | 形状 | 含义 |
|---|---|---|
| `observation/exterior_image_1_left` | `[224,224,3]` uint8 | 外部相机 |
| `observation/wrist_image_left` | `[224,224,3]` uint8 | 腕部相机 |
| `observation/joint_position` | `[7]` | 7 关节 |
| `observation/gripper_position` | `[1]` | 夹爪 |
| `prompt` | str | 语言指令 |

**模型内部处理**：`state = 关节[7] + 夹爪[1] = 8 维`，pad 到 32；图像补成 3 路（右腕缺→补零并 mask 掉）；输出 32 维动作**取前 8 维** = 7 关节 + 1 夹爪。

**两种落盘格式**：`LeRobot`（自定义小数据，几十小时内推荐）/ `RLDS`（完整 DROID 1.8TB 那种超大规模）。

## 6. 想改，改哪里

- 🟢 **超参 / LoRA**：`config.py` 里复制一个 `pi05_*` 条目改 `num_train_steps`、学习率、`action_horizon`；换 `gemma_2b_lora` 变体即低显存微调。
- 🟡 **接自己的机器人**：仿 `policies/droid_policy.py` 写平台 transform，动作/状态 pad 到 32，重算归一化统计量。改采样步数在 `pi0.py::sample_actions` 的 `num_steps`。
- 🔴 **改范式**：动作表示（flow matching ↔ FAST 离散 token）、时间注入（adaRMS ↔ 拼接 MLP）——改动面大，做研究用。

## 7. 一个真机坑

`examples/droid/main.py` 里硬断言动作块形状 `(10, 8)`、并用 **joint_velocity**；而 `pi05_droid` 是 `action_horizon=15`、官方推荐 **joint_position**。直接用 `pi05_droid` 起服务时，需把 `main.py` 的 `assert ...==(10,8)` 和 `open_loop_horizon` 相应改掉，否则形状不匹配。
